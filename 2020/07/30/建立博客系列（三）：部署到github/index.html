建立博客系列（三）：部署到github

发表于 2020-07-30 | 分类于 建立博客 | 阅读次数:

前面看到的只是本地效果，别人是看不到你写的博客的，因此我们要部署到github上，并使用自己的域名进行访问

一、在github上创建仓库

1、首先你要注册一个github账号

2、创建一个仓库

点击new repository

填写仓库名称blog，点击create repository创建仓库

二、添加SSH公钥

我们下载/克隆项目有两种方式https和SSH方式，https方式每次操作都需要输入github的账号和密码，不方便，所以我们采用SSH方式

1、打开git bash，输入如下命令创建SSH key

ssh-keygen -t rsa -C “your_email@youremail.com“

双引号里的邮箱改成你自己的邮箱，然后一路回车，会出现一张萌萌哒图

2、如果一切顺利的话，可以在用户主目录（C:\Users\用户名）里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。

3、打开github，点击setting

4、点击SSH and GPG keys，然后点击new SSH key

5、填写title和key

title是一个名称，随便输入一个就好

key填写id_rsa.pub文件的内容

三、域名

中间要插入一个话题，因为我们要使用自己的域名，所以我们来讲讲域名

1、首先要购买一级域名，如果不买的话访问链接就会很长很长，另外域名也不贵。

域名可以到腾讯云，阿里云购买，我是到腾讯云购买的

这里是部分域名的价格，我买的是最便宜的.top

如果大家准备购买的话，点开链接后注册账号即可，然后输入自己想购买的域名，再选择自己能接受的价格购买即可

购买之后一定要实名认证，有两处需要认证

认证官方说要3-7天，但我几个小时就认证成功了，可以先不管继续下面的操作

2、我们要配置域名解析，把你注册的域名解析到github上

点击解析

这里可以看到有两个默认的记录，不能删除哟

然后点击添加记录，添加两条记录

添加如下两条记录，把记录值改为自己的github用户名(小写).github.io，其他都一样。如果只添加第一条记录的话，我们将无法访问；如果只添加第二条记录的话，github上会报错。所以跟着一样就好。这里的添加和修改都需要10分钟才能生效，因为TTL是600秒

3、在你的博客目录的source下，我的是 E:\bolg\blog\source，创建一个文件，文件名为CNAME，必须是大写，没有后缀名。

打开，里面写你刚刚注册的域名，我的是www.gclstart.top
